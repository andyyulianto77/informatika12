Protokol apa yang harus digunakan aplikasi web Anda?

Apa itu subdomain? Anda memiliki domain, seperti silverthoughts.org. Namun, Anda ingin memiliki banyak situs web di dalamnya. Bisakah Anda melakukannya? 

Ya! Subdomain dapat membantu. Subdomain adalah nama yang diperluas, berdasarkan nama domain Anda, yang berfungsi seperti situs web terpisah. 

Ambil situs yang sedang Anda lihat sekarang. Situs itu ada di webapps.skilling.us. Itu adalah subdomain. 

Situs web ini adalah subproyek yang merupakan bagian dari Skilling. Saya membuat subdomain webapps, dan meletakkan berkas untuk situs web ini di sana. Ini adalah situs web terpisah dari yang ada di andyux.my.id .

Saya tidak perlu membayar untuk subdomain. Saya dapat membuat subdomain sebanyak yang saya inginkan, tanpa mengeluarkan uang sepeser pun. Hore! 

Yang lebih penting, situs-situs tersebut tidak saling mengganggu. Saya dapat mengutak-atik situs utama andyux.my.id (menambahkan file, menghapus file, apa pun), dan situs ini tidak akan terpengaruh.

Lebih, lebih, LEBIH BANYAK! 

Ada situs web kursus lain, tentang pemrograman Excel VBA. Itu ada di vba.skilling.us. Ada kursus lain tentang dasar-dasar web, di learnweb.skilling.us. 

Jadi ada empat situs terpisah: 

skilling.us:situs proyek utama 

learnweb.skilling.us: kursus dasar web 

vba.skilling.us: kursus pemrograman 

webapps.skilling.us:kursus ini 

(Sebenarnya ada lebih dari itu.) 

Subdomain memiliki folder di server Anda 

Setiap subdomain memiliki foldernya sendiri di server Anda. Begitulah cara subdomain dipisahkan. 

Berikut adalah folder untuk subdomain di atas. ~ adalah hal yang biasa di Linux. Itu adalah singkatan dari folder root akun pengguna saya.

learnweb.skilling.us

webapps.skilling.us

sk

folder server

~/public_html

~/learnweb.skilling.us

~/public.html

~/webapps.skilling.us

Pemetaan dari domain ( skilling.us) ke folder ~/public_html telah diatur oleh perusahaan hosting. Saya tidak mengubahnya. Subdomain, saya buat sendiri. Saya dapat menamai folder dengan nama apa pun yang saya inginkan (misalnya, anything-i-wanted), tetapi saya memberi folder dengan nama yang sama dengan subdomainnya, agar lebih mudah bagi saya. Mudah itu bagus.

Membuat subdomain 

Anda juga dapat memiliki subdomain di domain Anda. Misalnya, Anda dapat memiliki satu untuk proyek akhir Anda, dan satu lagi untuk latihan Anda yang lain. Memisahkan semuanya berarti file-file Anda tidak akan saling bertabrakan. 

Ayo kita lakukan.

Catatan

Jika Anda lupa cara membuat subdomain, kembali ke sini untuk mendapatkan pengingat.

Klik ikon Domain di beranda cPanel Anda, lalu buat yang baru. Ketik nama subdomain yang Anda inginkan untuk proyek akhir Anda, seperti final.ragingmonkeys.com atau final.kitsrule.net. 

cPanel akan menyarankan nama untuk Document Root (Lokasi Sistem Berkas). Di situlah berkas untuk subdomain Anda akan disimpan di server Anda. Kita akan membahasnya di pelajaran berikutnya. Jadikan root sama dengan nama domain Anda, agar lebih mudah, tetapi Anda dapat membuatnya dengan nama lain jika Anda suka. 

Jangan centang Bagikan akar dokumen. Inilah yang saya lakukan untuk membuat subdomain baru:

Root Dokumen adalah folder tempat file-file untuk subdomain Anda akan disimpan. Kita akan membahasnya di pelajaran berikutnya. Sekarang buat subdomain lain. Beri nama "exercises". Perhatikan akar dokumen. Ini akan menjadi seperti exercises.yourdomain.com. 

Latihan

Ringkasan 

  • Subdomain menambahkan sesuatu di bagian depan domain, seperti rosie.kid.com.
  • Subdomain berfungsi sebagai situs web terpisah. Buat sebanyak yang Anda inginkan, dari cPanel. Berikutnya Mari kita lihat bagaimana Anda mengacaukan berkas pada server Anda.

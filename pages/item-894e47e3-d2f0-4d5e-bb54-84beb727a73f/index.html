Protokol apa yang harus digunakan aplikasi web Anda?

Apa itu subdomain? Anda memiliki domain, seperti silverthoughts.org. Namun, Anda ingin memiliki banyak situs web di dalamnya. Bisakah Anda melakukannya? 

Ya! Subdomain dapat membantu. Subdomain adalah nama yang diperluas, berdasarkan nama domain Anda, yang berfungsi seperti situs web terpisah. 

Ambil situs yang sedang Anda lihat sekarang. Situs itu ada di webapps.skilling.us. Itu adalah subdomain. 

Situs web ini adalah subproyek yang merupakan bagian dari Skilling. Saya membuat subdomain webapps, dan meletakkan berkas untuk situs web ini di sana. Ini adalah situs web terpisah dari yang ada di andyux.my.id .

Saya tidak perlu membayar untuk subdomain. Saya dapat membuat subdomain sebanyak yang saya inginkan, tanpa mengeluarkan uang sepeser pun. Hore! 

Yang lebih penting, situs-situs tersebut tidak saling mengganggu. Saya dapat mengutak-atik situs utama andyux.my.id (menambahkan file, menghapus file, apa pun), dan situs ini tidak akan terpengaruh.

Lebih, lebih, LEBIH BANYAK! 

Ada situs web kursus lain, tentang pemrograman Excel VBA. Itu ada di vba.skilling.us. Ada kursus lain tentang dasar-dasar web, di learnweb.skilling.us. 

Jadi ada empat situs terpisah: 

skilling.us:situs proyek utama 

learnweb.skilling.us: kursus dasar web 

vba.skilling.us: kursus pemrograman 

webapps.skilling.us:kursus ini 

(Sebenarnya ada lebih dari itu.) 

Subdomain memiliki folder di server Anda 

Setiap subdomain memiliki foldernya sendiri di server Anda. Begitulah cara subdomain dipisahkan. 

Berikut adalah folder untuk subdomain di atas. ~ adalah hal yang biasa di Linux. Itu adalah singkatan dari folder root akun pengguna saya. 

 ~/public_html learnweb.skilling.us ~/learnweb.skilling.us  ~/vba.skilling.us webapps.skilling.us ~/webapps.skilling.us Pemetaan dari domain ( skilling.us) ke folder ~/public_htmltelah diatur oleh perusahaan hosting. Saya tidak mengubahnya. Subdomain, saya buat sendiri. Saya dapat menamai folder dengan nama apa pun yang saya inginkan (misalnya, anything-i-wanted), tetapi saya memberi folder dengan nama yang sama dengan subdomainnya, agar lebih mudah bagi saya. Mudah itu bagus.

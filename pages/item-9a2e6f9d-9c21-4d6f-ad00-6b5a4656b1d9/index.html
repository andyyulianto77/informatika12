Anda mengetik URL ke dalam browser. Seperti https://andyux.my.id, atau http://andy1.rf.gd/kucing/kity.html. Apa itu URL?

URL adalah alamat

Setiap berkas di Web memiliki URL (uniform resource locator). Baik itu berkas HTML, berkas foto, atau apa pun, berkas tersebut memiliki URL. URL berkas adalah alamat uniknya di Web.

Coba URL ini: http://andy1.rf.gd/kucing/kity.html

URL memiliki tiga bagian:

  • Sebuah protokol:http
  • Domain: http://andy1.rf.gd. (Sebenarnya itu adalah subdomain. Nanti akan dijelaskan lebih lanjut.)
  • Jalannya: /kucing/kity.html

Protokol

Protokol adalah serangkaian aturan tentang bagaimana klien (seperti browser Anda) dan server berkomunikasi satu sama lain. Saat browser Anda menginginkan sebuah berkas, ia akan mengirimkan kata tersebut GET ke server. Saat server menerima kata tersebut GET, ia tahu bahwa itu adalah permintaan berkas. 

Mengapa kata GET, dan bukan SENDME, atau BEKOMMEN? Tidak ada alasan yang tepat, sungguh. Seorang pria Inggris memilih kata itu GETbertahun-tahun yang lalu. Selama komputer sepakat tentang kata apa yang digunakan, tidak masalah kata apa yang digunakan. 

Orang yang memilih GETmemberi nama protokol HTTP, yang merupakan singkatan dari... Anda dapat mencarinya, jika Anda peduli. 

Berikut URL-nya lagi: 

http://andy1.rf.gd/kucing/kity.html

Katanya "http", tetapi ada juga "https". Huruf S di akhir berarti "aman." Permintaan dari peramban Anda ke server dienkripsi, seperti mata-mata rahasia. Respons dari server ke peramban Anda juga dienkripsi. 

Saat ini, Anda biasanya harus menggunakan https. Sejauh pengetahuan saya, tidak ada alasan yang kuat untuk tidak melakukannya.

Wilayah kekuasaan 

URL-nya adalah:

http://andy1.rf.gd/kucing/kity.html

andy1.rf.gd adalah domain . Itu seperti nama server web.

Untuk menaruh sesuatu di web, Anda memerlukan dua hal: 

  • Server untuk menaruh berkas.
  • Nama untuk server tersebut, yaitu domain.
Ketika Anda membeli hosting dari Reclaim, Anda mendapatkan dua hal: 

  • Ruang di server untuk menaruh berkas.
  • Nama yang menunjuk ke ruang server Anda.
Nama domain Anda sebenarnya adalah tempat penampung untuk alamat IP. Nanti akan dibahas lebih lanjut. 

Jalan setapak

URL lagi:

http://webappexamples.skilling.us/dogs/renata.html

Bit pertama http://webappexamples.skilling.us mengidentifikasi server.

Bit terakhir, /dogs/renata.html, adalah path . Ini adalah lokasi file di server.

Pikirkan komputer Anda, yang Anda gunakan untuk membaca teks ini. Jika Anda menggunakan ponsel atau tablet, itu adalah komputer. Komputer kecil, tetapi tetap saja komputer.

Komputer Anda memiliki ribuan file. File-file tersebut dikelompokkan ke dalam folder. Windows, Mac OS, Linux, Android... semuanya menyimpan file dalam folder.

("Folder" dan "direktori" memiliki arti yang sama.)

/dogs/renata.html memberitahu server untuk masuk ke folder dogs, dan mencari file bernama renata.html.

(Baiklah, itu dapat diatur secara berbeda, tetapi jalankan saja untuk saat ini.)
